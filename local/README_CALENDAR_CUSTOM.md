# Инструкция по установке и настройке

## Описание
Данная доработка добавляет в календарь Битрикс24 (коробочная версия) обязательный выбор типа события и поле выбора компании для типа "Встреча".

## Типы событий
- Встреча
- Собеседование
- Созвон
- Другое

## Структура файлов

```
local/
├── modules/
│   └── calendar/
│       ├── include.php              # Обработчики событий
│       └── .description.php         # Описание модуля
├── components/
│   └── custom/
│       └── calendar.event.edit/
│           ├── class.php            # Класс компонента
│           ├── component.php        # Логика компонента
│           ├── template.php         # Шаблон компонента
│           ├── .description.php     # Описание компонента
│           ├── .parameters.php      # Параметры компонента
│           └── ajax.php             # AJAX обработчик
└── js/
    └── calendar/
        └── event_fields.js          # JavaScript для интеграции
```

## Установка

### 1. Скопируйте файлы
Скопируйте все файлы из папки `local` в корень вашего сайта Битрикс24.

### 2. Подключите обработчики событий
Добавьте в файл `/bitrix/php_interface/init.php` следующий код:

```php
<?php
require_once($_SERVER["DOCUMENT_ROOT"]."/local/modules/calendar/include.php");
```

Если файла `init.php` не существует, создайте его.

### 3. Подключите JavaScript
Добавьте подключение JavaScript файла в шаблон сайта или в настройки календаря.

**Вариант А:** В файл `/bitrix/php_interface/init.php`:

```php
\Bitrix\Main\Page\Asset::getInstance()->addJs('/local/js/calendar/event_fields.js');
```

**Вариант Б:** В шаблон сайта (`header.php`):

```php
$APPLICATION->SetAdditionalJS('/local/js/calendar/event_fields.js');
```

### 4. Настройка прав доступа
Убедитесь, что у пользователей есть права на:
- Модуль "Календарь"
- Модуль "CRM" (для доступа к компаниям)

## Как это работает

### При создании события
1. Пользователь открывает форму создания события в календаре
2. Автоматически добавляются два новых поля:
   - **Тип события** (обязательно) - выпадающий список
   - **Компания** (обязательно только для типа "Встреча") - выпадающий список с компаниями из CRM

3. При выборе типа "Встреча" появляется поле выбора компании
4. При попытке сохранить событие без заполнения обязательных полей выводится предупреждение

### После сохранения
Данные о типе события и компании сохраняются в описании события в специальном формате:

```
[CUSTOM_DATA]
EVENT_TYPE:meeting
COMPANY_ID:123
[/CUSTOM_DATA]
```

### При редактировании
При открытии формы редактирования существующего события:
- Тип события и компания автоматически выбираются из сохраненных данных
- Поле компании показывается только если тип события - "Встреча"

## Получение данных программно

Для получения данных о типе события и компании используйте функцию:

```php
$eventData = CustomCalendar_GetEventData($description);
// Возвращает массив:
// [
//     'event_type' => 'meeting',
//     'company_id' => 123,
//     'company_name' => 'Название компании'
// ]
```

## Отчеты и списки

Для отображения типа события и компании в списках и отчетах можно использовать регулярное выражение для парсинга поля DESCRIPTION:

```php
preg_match('/EVENT_TYPE:(\w+)/', $description, $matches);
$eventType = $matches[1] ?? '';
```

## Возможные проблемы и решения

### Поля не появляются в форме
1. Проверьте, что JavaScript файл подключен
2. Откройте консоль браузера (F12) и проверьте наличие ошибок
3. Убедитесь, что форма календаря загружается полностью

### Не загружается список компаний
1. Проверьте, что модуль CRM установлен и активен
2. Проверьте права доступа пользователя к компаниям CRM
3. Проверьте консоль браузера на ошибки AJAX запросов

### Данные не сохраняются
1. Проверьте, что обработчики событий подключены через `init.php`
2. Проверьте логи Битрикс на наличие ошибок
3. Убедитесь, что у события есть поле DESCRIPTION

## Дополнительная настройка

### Изменение типов событий
Отредактируйте файл `template.php` и `event_fields.js`, изменив опции в select:

```html
<option value="meeting">Встреча</option>
<option value="interview">Собеседование</option>
...
```

### Добавление поиска компаний
В текущей реализации поиск уже реализован. Для улучшения можно добавить Select2 или другой UI компонент.
